## 🧹 重大体验改进：「手动输入模型名」输入框全面下线

### 痛点回顾

旧版本里，几乎每个会用到 AI 模型的地方都有一个**「或手动输入模型名」**的备用输入框：

| 位置 | 旧 DOM ID/Class | 出现原因 |
|------|----------------|---------|
| ① 普通对话顶部 | `#custom-model` | 应对模型列表不全 |
| ② AI 对战参与者卡片 | `.pc-custom-model` | 同上 |
| ③ 思维导图左侧 | `#mm-custom-model` | 同上 |
| ④ 工作流 AI 节点 | `<input df-custom-model>` | 同上 |
| ⑤ 图像分析任务卡（新增） | 任务卡内 input | 同上 |
| ⑥ 图像分析任务卡（恢复） | restoreAnalyzeTask 同上 | 同上 |
| ⑦ 一键3D建模 Step 2 节点 | `.m3d-step-custommodel[data-step=2]` | 同上 |
| ⑧ 一键3D建模 Step 3 节点 | `.m3d-step-custommodel[data-step=3]` | 同上 |
| ⑨ 一键3D建模 Step 4 节点 | `.m3d-step-custommodel[data-step=4]` | 同上 |

> **总共 9 处冗余 input**（v1.6.2 的统计是 8 处，实际包含图像分析的 add+restore 两份模板，合并算 8 处场景、9 个 DOM 节点）。

用户反馈：**"每个地方都问我要输模型名，到底应该填还是从下拉框选？为什么有的填了下拉框就没了？"**

### 新方案

**唯一入口**：设置 → 已保存的 API → 点击「筛选模型」按钮 → 弹窗底部「➕ 手动添加自定义模型 ID」折叠区。

- 添加 `gpt-image-2` / `claude-opus-4` / `dall-e-3` 等 `/v1/models` 不返回的模型
- 添加后该模型会出现在弹窗模型列表顶部（标记「自定义」），默认勾选
- 保存筛选后，**所有 5 个模式（普通对话/AI 对战/思维导图/工作流/摄影 5 步）** 的下拉框都能选到这个模型
- 后端 `customModels` 字段持久化，重启不丢

### 截图操作流程

```
点击「设置」按钮
   ↓
找到已保存的 API（如「咸鱼 API」）
   ↓
点击「筛选模型」按钮
   ↓
展开底部「➕ 手动添加自定义模型 ID」
   ↓
输入 gpt-image-2 → 点「添加」
   ↓
点「保存筛选」
   ↓
✅ 完成！所有模式的下拉框立即可用
```

## 🚦 3D 建模运行中切换防误删

旧版本里如果一边在跑一键3D建模 Step 2，一边切换到普通对话模式（或切到其他摄影项目），会导致：

- AbortController 没被取消，请求继续在后台跑
- 但返回结果会写到错误的 m3dState 里（因为切换后状态已重置）
- 已经生成的 SceneSummary / 四宫格图 / SceneSpec 被新加载的项目数据覆盖

**v1.6.3 修复**：

- `m3dIsAnyRunning()` 检查 Step 2/3/4 任一节点是否在跑
- 切换模式（`switchMode`）前弹确认框：
  > ⚠️ 一键3D建模流水线还有节点正在生成中，切换模式会取消正在运行的请求（已生成的结果不会丢失）。是否继续？
- 切换摄影项目（`selectPhotoSession`）前弹同样确认框
- 确认后自动调用 `m3dCancelAll()` 取消所有正在运行的 step，再正常切换

## 🎯 Step 2 提示词：环境与人物彻底分家

旧版本 Step 2 总结场景时，AI 经常这样写：

> 「...场景中央有一位身着白色连衣裙的少女正在举着泡泡机轻轻挥手，她的眼神望向远方海平面...」

但后续的 3D 建模流程**根本不需要人物**，这些描写会：

1. 让 Step 3 的四宫格图里出现不必要的人物（占用画面，分散 SceneSpec 关注点）
2. 让 Step 4 测算 SceneSpec 时把人物当成物体放进 objects 列表
3. Step 5 生成的 bpy 脚本里会出现奇怪的"人物 cube"

**v1.6.3 修复**（强化 System Prompt）：

```
【只总结环境与场景，绝对不要输出任何人物 / 动物 / 生物】：
- 不要描写人物的外貌、表情、动作、姿势、服装、年龄、性别
- 不要描写人物在场景里"做什么"
- 也不要描写宠物、动物、虚拟角色等任何有生命的主体
- 即使对话里出现人物，也只提取他们所处的"环境"
```

同时强化"必须输出散文，禁止 JSON / Markdown 列表 / 标题 / 代码块"，避免回退到 v1.6.2 之前的 JSON 失败。

## 🐛 顺带修复

- **`package.json` 重复 `name` 字段**：原本同时存在 `"name": "ai-multi-chat"` 和 `"name": "ai-chat-app"`，JSON 不报错但 npm 会忽略前者。本次合并为唯一字段 `ai-chat-app`，版本号同步升至 `1.6.3`。
- **AI 对战 / 图像分析的"未选模型"错误提示**：新增引导文案"未找到模型时可去 设置 → API → 筛选模型 → 手动添加自定义模型 ID"。

## 🔄 向后兼容性

完全向后兼容 v1.6.x 之前的所有项目数据：

| 场景 | 行为 |
|------|------|
| 老工作流 AI 节点保存了 `customModel` 字段 | selectWfSession 恢复时如果保存的 model 不在筛选列表里，会自动把 customModel 加到下拉框（标「(自定义)」）并选中 |
| 工作流运行时上下游图生图，节点只有 `customModel` 没有 `model` | `runWorkflow` 优先用 `model`，没有则回落 `customModel` |
| 老摄影项目图像分析任务的 `customModel` | 静默保留为空字符串，不影响读取 |
| 普通对话 / AI 对战 / 思维导图保存的 `customModel` | 已经从未读取过（v1.6.2 之前的代码就没用过），直接静默忽略 |

## 🚀 升级指南

### 普通用户

1. 直接下载新版安装包覆盖安装即可
2. 首次启动会弹出 v1.6.3 欢迎弹窗，列出本次更新内容
3. 所有 API 配置、会话历史、摄影项目数据完全保留

### 添加自定义模型 ID（如 gpt-image-2）

1. 点击侧边栏「设置」
2. 找到要配置的 API（如咸鱼 API）→ 点「筛选模型」
3. 展开底部「➕ 手动添加自定义模型 ID」
4. 输入模型 ID（例如 `gpt-image-2`）→ 点「添加」
5. 列表里会出现新模型（默认勾选）→ 点「保存筛选」
6. 在任何用到模型的地方（普通对话/工作流/3D建模 Step 3）的下拉框就能选到了 ✅

### 老工作流项目（含 customModel 字段）

无需任何操作，打开项目时会自动兼容显示。如果想清理掉「(自定义)」标签：

1. 进入「设置 → API → 筛选模型 → 手动添加自定义模型 ID」
2. 把对应的模型 ID 添加进去
3. 重新打开老工作流，AI 节点的下拉就会显示正常的模型名（不再带「(自定义)」后缀）
