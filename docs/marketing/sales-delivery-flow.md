# Sales Delivery Flow

## No-Cloud Launch Flow

This product can be sold without cloud accounts or a license server.

1. Publish website or sales page.
2. Buyer reads requirements and sees screenshots/video.
3. Buyer pays through a payment link.
4. Buyer receives download link and setup documentation.
5. Buyer installs the macOS DMG.
6. Buyer installs Blender bridge plugin.
7. Buyer configures their own API provider.
8. Buyer follows the first-scene tutorial.

## Suggested Sales Channels

China-focused:

- 微信 / 支付宝收款 + 飞书表单.
- 小报童 for paid posts/tutorials.
- 百度网盘 / 阿里云盘 / 123网盘 for file delivery.
- B站 / 小红书 / 抖音 for demo traffic.

International-ready later:

- Gumroad.
- Lemon Squeezy.
- Ko-fi.
- GitHub Releases for free assets only, not private paid builds.

## Delivery Package

Minimum paid delivery package:

- `白歌的AI讨论组-3.7.0-arm64.dmg`
- `白歌的AI讨论组-3.7.0-x64.dmg`
- Setup PDF or web page.
- Blender plugin install guide.
- First-scene tutorial.
- Third-party notices.
- Refund/support policy.

## Manual Fulfillment Template

Message to buyer:

> 感谢购买白歌的AI讨论组专业版。下载链接如下：
>
> - macOS Apple Silicon: 发送当前版本 arm64 DMG 下载地址
> - macOS Intel: 发送当前版本 x64 DMG 下载地址
> - 安装教程: 发送当前版本安装教程地址
>
> 使用前请准备：Blender、自己的 AI API Key。3D 功能需要安装 Blender 插件。首次安装遇到问题可以把截图和系统版本发给我。

## Update Flow

For early sales, updates can be delivered manually:

1. Build new DMG.
2. Update changelog.
3. Upload to delivery folder.
4. Notify paid buyers.

Before scaling, consider:

- Signed builds.
- Auto-update.
- License key system.
- A proper storefront.
